# Stage 1: Markdown → ガントチャート YAML 変換手順

Claude Code がこの手順書に従って、雑な Markdown をガントチャート用 YAML に変換する。

---

## 概要

**入力**: 企画メモ・プロジェクト計画などの自由記述 Markdown ファイル  
**出力**: `docs/spec-format.md` に従った YAML ファイル  
**使用ツール**: Claude Code の Read / Write ツール

---

## 実行手順

### Step 1: 入力ファイルを読む

```
Read: <入力Markdownファイルのパス>
```

### Step 2: 情報を抽出する

以下を読み取る。不明な情報は後述のデフォルト・推測ルールで補う。

1. **プロジェクト名** — タイトルや見出しから
2. **全体期間** — 明示されていれば取得。なければタスクの日付から推定
3. **フェーズ・カテゴリ** — 大見出し (`##`) や「フェーズ1」などのグループ分け
4. **タスク一覧** — 各フェーズ内の作業項目
5. **マイルストーン** — 「完了」「承認」「リリース」などの節目イベント
6. **依存関係** — 「〜完了後」「〜が終わったら」「〜と並行」などの表現

### Step 3: 曖昧な日付・期間を解釈する

今日の日付: `currentDate` システム変数を参照すること。

#### 絶対日付の解釈
| 表現例 | 解釈 |
|--------|------|
| `6月1日`, `6/1` | その年の6月1日 |
| `6月第1週` | その月の第1月曜日 |
| `6月中旬` | 6月11〜20日 → 6月15日として扱う |
| `6月末` | 6月30日 |
| `2026年Q2` | 2026-04-01 〜 2026-06-30 |

#### 相対表現の解釈
| 表現例 | 解釈 |
|--------|------|
| `来月` | 今日の翌月1日 |
| `今月末` | 今月の最終日 |
| `今週中` | 今週金曜日 |
| `〜から開始` | 直前のタスク終了日の翌営業日 |

#### 期間の解釈
| 表現例 | 日数換算 |
|--------|---------|
| `1週間` | 7日 |
| `約2週間`, `2週間程度` | 14日 |
| `1ヶ月`, `約1ヶ月` | 30日 |
| `2ヶ月` | 60日 |
| `数日` | 3日 |
| `半月` | 15日 |

#### 並行・依存の解釈
- 「〜と並行して」→ 同じ start 日、dependencies には入れない（またはどちらか先に始まる方を依存元に）
- 「〜完了後」「〜が終わったら」→ dependencies に追加、start = 依存先の end 翌日
- 「〜の後すぐ」→ dependencies に追加、1〜2日のラグ
- 「〜が始まってから」→ dependencies に追加、start を依存元の start と同日 or 数日後

#### 期間が書かれていない場合
- 会議・MTG・レビュー → 1〜2日
- 設計・調査・計画 → 1〜2週間
- 開発・実装 → 3〜6週間（文脈による）
- テスト → 1〜2週間
- デプロイ・リリース → 1〜3日

### Step 4: YAML を生成する

- フォーマットは `docs/spec-format.md` に完全に従う
- ID は `t01`, `t02`, ... / `m1`, `m2`, ... / `phase1`, `phase2`, ... とする
- タスクは group ごとにまとめ、start 日順に並べる
- `display.unit` はプロジェクト全体期間を見て選ぶ（3ヶ月以内 → week、それ以上 → month）
- 解釈に迷った点は `note` フィールドに記録する
- **推測した内容は note に「（推測）」と明記する**

### Step 5: ファイルに書き出す

出力ファイルのパスは入力ファイル名を元に決める。

例:
- 入力: `sample/my-project.md`
- 出力: `data/my-project.yaml`

```
Write: <出力YAMLファイルのパス>
```

---

## 実行例

ユーザーが「`sample/my-project.md` をガントチャートにして」と言ったら:

1. `Read: sample/my-project.md` で内容を読む
2. 上記 Step 2〜4 に従って YAML を組み立てる
3. `Write: data/my-project.yaml` で出力する
4. 「`data/my-project.yaml` を生成しました。`app/index.html` をブラウザで開いてアップロードしてください。」と伝える

---

## 品質チェックリスト

出力前に確認:
- [ ] 全タスクに `start` と `end` がある
- [ ] `start` ≤ `end` になっている
- [ ] `dependencies` で参照している id が実在する
- [ ] `group` で参照している id が `groups` に定義されている
- [ ] マイルストーンの `date` が妥当な範囲内にある
- [ ] `display.start` ≤ 最初のタスク開始日、`display.end` ≥ 最後のタスク終了日
